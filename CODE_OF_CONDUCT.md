# Code of Conduct

This project is intended for practical, respectful collaboration around security-sensitive infrastructure.

## Expected Behavior

- Be clear, factual, and respectful.
- Keep discussions focused on reproducible technical details.
- Do not publish secrets, private endpoints, client configs, subscription URLs, or exploit details in public threads.
- Assume good intent, but prioritize user safety and infrastructure security.

## Unacceptable Behavior

- Harassment, personal attacks, or discriminatory language.
- Publishing credentials, private keys, live server data, or doxxing material.
- Sharing exploit instructions against real systems outside a responsible disclosure flow.

## Reporting

Use private vulnerability reporting for security issues. Public issues should contain only sanitized technical information.
