## 1. Что это за проект сейчас

Проект уже не является просто bootstrap-заготовкой. На текущем этапе это рабочий single-node control plane для `Xray-core + VLESS + REALITY`, состоящий из:

- `api`: NestJS + TypeScript + Prisma + PostgreSQL
- `web`: React + TypeScript + Vite
- `xray`: data plane на `443/tcp`
- `caddy`: reverse proxy панели на `8443/tcp`
- host-level `ufw` и `fail2ban`

Архитектурная идея:

- `443` зарезервирован под VPN-трафик Xray
- панель живет на `8443`, потому что домен под публичный TLS для панели пока не задан
- `Xray` и `api` связаны через внутреннюю control network
- PostgreSQL является каноническим control-plane хранилищем
- конфиг Xray не редактируется руками на проде, а рендерится из шаблона

После включения file logs Xray падал из-за permission denied на host bind mount.

Исправление:

- `c1fbbd6` `fix: prepare runtime log files before deploy`

### 4.5. Basic auth vs Bearer conflict

Был важный баг: edge `basic_auth` в Caddy стоял на `/api/*`, а frontend использует `Bearer` для app auth. Это делало браузерную панель логически сломанной.

Исправление:

- `aa75c47` `fix: bypass edge basic auth for api routes`

Итоговая модель:

- UI root и статика по-прежнему защищены basic auth
- `/api/*` идет без edge basic auth
- API защищен приложением: login + bearer + refresh flow

### 4.6. Legacy BACKUP_DIR migration

На сервере был старый `BACKUP_DIR` с host path вместо container path, из-за чего backup creation отдавал `500`.

Исправление:

- `51fe1ab` `fix: normalize legacy backup path in deploy`

`deploy.sh` теперь умеет автоматически нормализовать legacy `BACKUP_DIR` в `.env`.
